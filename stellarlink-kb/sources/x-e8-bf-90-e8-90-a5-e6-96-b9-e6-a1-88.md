## [一、算法核心逻辑速览](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#%E4%B8%80%E7%AE%97%E6%B3%95%E6%A0%B8%E5%BF%83%E9%80%BB%E8%BE%91%E9%80%9F%E8%A7%88)

```
候选召回 → 硬过滤 → 多目标预测 → 加权合分 → 多样性衰减 → TopK选取 → 安全过滤
```

**决定推荐的三个关卡**：

1.  能否进入候选池（in-network 优势巨大）
2.  会否被硬过滤（过期/违规/已看等）
3.  engagement 加权总分高低

### [3.1 内容结构设计原则](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#31-%E5%86%85%E5%AE%B9%E7%BB%93%E6%9E%84%E8%AE%BE%E8%AE%A1%E5%8E%9F%E5%88%99)

#### [开头钩子（决定 Dwell 信号）](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#%E5%BC%80%E5%A4%B4%E9%92%A9%E5%AD%90%E5%86%B3%E5%AE%9A-dwell-%E4%BF%A1%E5%8F%B7)

```
✅ 推荐结构：
- 争议观点开头："大多数人都错了..."
- 数据/事实开头："90%的人不知道..."
- 问题开头："为什么 X 比 Y 更难..."
- 反直觉开头："我曾经也这么认为，直到..."

❌ 避免：
- 泛泛而谈的开头
- 需要翻页才能看到价值的内容
- 纯表情/表情包开头
```

#### [正文结构（提升停留时长）](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#%E6%AD%A3%E6%96%87%E7%BB%93%E6%9E%84%E6%8F%90%E5%8D%87%E5%81%9C%E7%95%99%E6%97%B6%E9%95%BF)

```
✅ 推荐：
- 分段清晰，每段 2-3 行
- 使用列表/编号
- 关键信息加粗或用 emoji 标记
- 结尾有明确的行动召唤

❌ 避免：
- 大段文字堆砌
- 没有视觉断点
- 信息密度过低
```

## [四、发布节奏策略](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#%E5%9B%9B%E5%8F%91%E5%B8%83%E8%8A%82%E5%A5%8F%E7%AD%96%E7%95%A5)

### [4.1 频率控制](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#41-%E9%A2%91%E7%8E%87%E6%8E%A7%E5%88%B6)

```
⚠️ 同作者多样性衰减机制
代码位置：author_diversity_scorer.rs:29
同一作者在单次推荐请求中出现越多，后续内容衰减越严重

✅ 推荐频率：
- 日常：1-2 条/天
- 活跃期：最多 3-4 条/天
- 间隔：至少 2-4 小时

❌ 避免：
- 短时间内连续发布（刷屏）
- 一天发布 5+ 条
- 集中发布后长期沉默
```

### [4.2 时间选择](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#42-%E6%97%B6%E9%97%B4%E9%80%89%E6%8B%A9)

```
原则：根据目标用户活跃时间发布

通用建议（需根据实际数据调整）：
- 工作日：8-9am, 12-1pm, 6-8pm
- 周末：10-11am, 3-5pm

⚠️ 注意：
- 内容有过期时间窗口（age_filter.rs:8）
- 在网召回按时间倒序（thunder_service.rs:333）
- 新鲜内容天然有优势
```

### [5.1 为什么涨粉是核心](https://stellarlink.co/articles/x%E8%BF%90%E8%90%A5%E6%96%B9%E6%A1%88#51-%E4%B8%BA%E4%BB%80%E4%B9%88%E6%B6%A8%E7%B2%89%E6%98%AF%E6%A0%B8%E5%BF%83)
